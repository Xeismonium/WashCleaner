# Milestone v1.1 Requirements: Data & Domain Layer

## Domain Models (DOM-MOD)
- **DOM-MOD-01**: Implement `User`, `Order`, `Customer`, `Service`, and `StoreSettings` data classes.
- **DOM-MOD-02**: Ensure all fields are non-nullable with sensible defaults.

## Repository Interfaces (DOM-REP)
- **DOM-REP-01**: `AuthRepository` (Login, Logout, Session, Roles).
- **DOM-REP-02**: `OrderRepository` (CRUD, Status updates, filtering, paging support).
- **DOM-REP-03**: `CustomerRepository` (CRUD, Search).
- **DOM-REP-04**: `ServiceRepository` (CRUD, Activation toggles).
- **DOM-REP-05**: `SettingsRepository` (Store Info, Staff Management).

## Use Cases (DOM-UC)
- **DOM-UC-01**: `GetOrdersUseCase`, `CreateOrderUseCase`, `UpdateOrderStatusUseCase`.
- **DOM-UC-02**: `GenerateOrderCodeUseCase` (Firestore-backed counter).
- **DOM-UC-03**: `GetReportSummaryUseCase` (Aggregation).

## Local Data (LOC-DATA)
- **LOC-DATA-01**: Room entities mapped from domain models (`OrderEntity`, `CustomerEntity`, `ServiceEntity`).
- **LOC-DATA-02**: TypeConverters for `Timestamp` (stored as `Long`).
- **LOC-DATA-03**: DAOs for all entities with required queries (filtering, searching, WorkManager support).
- **LOC-DATA-04**: `AppDatabase` singleton implementation.

## Remote Data (REM-DATA)
- **REM-DATA-01**: `FirestoreDataSource` for raw read/write operations.
- **REM-DATA-02**: Enforce `createdAt` and `updatedAt` server timestamps on all writes.
- **REM-DATA-03**: Return `Result<T>` for all network operations.

## Synchronization & Persistence (SYNC)
- **SYNC-01**: Repository implementations using offline-first strategy (Room write -> Firestore sync).
- **SYNC-02**: Real-time updates from Firestore to Room via snapshot listeners.
- **SYNC-03**: `SessionManager` using `DataStore Preferences` for persistent user session.

## Infrastructure (INF)
- **INF-08**: Hilt modules for `AppModule`, `FirebaseModule`, `DatabaseModule`, and `RepositoryModule`.
- **INF-09**: Core utilities: `CurrencyFormatter` (Rupiah), `DateFormatter` (Indonesian locale), `OrderCodeGenerator`.
