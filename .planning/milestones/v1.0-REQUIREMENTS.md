# v1 Requirements: WashCleaner

## Authentication & Roles (AUTH)
- **AUTH-01**: User can log in with email and password.
- **AUTH-02**: Role-based access control (Owner vs Staff).
- **AUTH-03**: App checks `isActive` status before allowing login.
- **AUTH-04**: Staff session is persisted locally for offline use.

## Customer Management (CUST)
- **CUST-01**: User can create, read, and update customer profiles.
- **CUST-02**: Search customers by name or phone number.
- **CUST-03**: View customer order history.

## Order Management (ORD)
- **ORD-01**: User can create new orders (Kiloan/weight-based and Satuan/item-based).
- **ORD-02**: Auto-generate unique order codes (format: WC-YYYYMMDD-XXX).
- **ORD-03**: Update order status (Received, Washing, Ironing, Ready, Done).
- **ORD-04**: Estimate finish date based on store defaults or manual selection.
- **ORD-05**: Soft delete orders (Owner only).

## Cashier & Payment (PAY)
- **PAY-01**: Record payments (Paid, Unpaid, Down Payment).
- **PAY-02**: Calculate total price based on service rates and weight.
- **PAY-03**: Send receipts via WhatsApp using Android Intents.
- **PAY-04**: Track remaining balance for partial payments (Down Payment).

## Services & Settings (SET)
- **SET-01**: Owner can manage service rates (Name, Price per Kg/Unit).
- **SET-02**: Owner can manage store info (Name, Phone, Hours).
- **SET-03**: Owner can manage staff accounts (Add, Activate/Deactivate).

## Reporting (REP)
- **REP-01**: Owner can view daily, weekly, and monthly revenue reports.
- **REP-02**: Visualize revenue trends using bar charts (Vico).
- **REP-03**: View list of unpaid orders.

## Infrastructure (INF)
- **INF-01**: Offline-first operation using Room database.
- **INF-02**: Background sync with Firestore using WorkManager.
- **INF-03**: Local notifications for pickup reminders (orders ready > 3 days).
- **INF-04**: Centralized dependency management via Version Catalog (`libs.versions.toml`).
- **INF-05**: Dependency Injection setup with Hilt.
- **INF-06**: Base UI system with Material 3, custom theme, and status-specific color constants.
- **INF-07**: Complete project skeleton with clean architecture layers and placeholder files.

---

## v2 Requirements (Deferred)
- **v2-01**: Expense tracking (Operational costs like detergent, rent).
- **v2-02**: QR/Barcode tagging for items.
- **v2-03**: Priority/Express order logic.
- **v2-04**: Bluetooth thermal printer support.

## Out of Scope
- **OOS-01**: Payment Gateway (Manual recording + WA suffices).
- **OOS-02**: Multi-outlet support.
- **OOS-03**: Customer-facing app.
